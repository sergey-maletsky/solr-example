Версия Java: 1.8.0_131

Инструкция по развертыванию:

I Общая информация

1. Приложение работает на 7070 порту
2. Swagger Api документация доступна по адресу http://localhost:7070/swagger-ui.html

II Настройка Solr

1. перейти по ссылке http://www.apache.org/dyn/closer.lua/lucene/solr/7.4.0
2. скачать архив Solr версии 7.4.0 и распаковать
4. перейти в <распакованный архив solr>/bin
5. в терминале выполнить "./solr start" (Linux) или "solr.cmd start" (Win)
6. затем выполнить "./solr create -c users" и "./solr create -c amounts" (Linux)
или "solr.cmd create -c users" и "solr.cmd create -c amounts" (Win)

III Запуск проекта

Для того, чтобы собрать и развернуть проект, необходимо:
1. выполнить из терминала ./run из корня проекта
2. в браузере открыть http://localhost:7070/

IV Описание работы приложения

С помощью реализации solr на spring data, документы при сохранении сразу попадают в индекс,
что предоставляет очень быстрый поиск по индексу.
В тестовой реализации доступно сгенерировать 5 млн. пользователей и производить поиск по ним, а также
добавлять пользователей по одному.
После индексирования можно искать по тексту, например, из тестовых данных мы можем отобрать
всех пользователей с префиксом в имени De, который отделен пробелом от остальных частей имени или
по окончанию имени, например, Rudo.

Также реализована возможность поиска по нечетким запросам(fuzzy search).
Для поиска только по имени(с пустым полем возраста) редакционное расстояние равно 1,
а при поиске по имени и возрасту редакционное расстояние равно 2.
Т.е. в первом случае можно сделать только одну ошибку в поиске, т.о. если мы ищем пользователя
с окончанием имени на Niro, то найти удастся таких пользователей делая запросы, например, iro или
Nird или Nro и др.

Во втором случае, если мы ищем пользователей с именем начинающимся на Robert и возрасту, то,
сделав ошибку при вводе имени, например, Robeld или Loberd или orbert или даже ober,
система найдет именно пользователей с именем Robert.
